# Know Where To Do

이 문서는 [PRD.md](/E:/Google%20Drive/VIBE_class/Know_where/PRD.md)를 실제 구현으로 전환하기 위한 작업 목록이다.  
순서는 선행 의존성을 반영하며, 각 단계는 완료 기준이 명확해야 한다.

## 1. 진행 원칙

- UI 구조는 PRD의 UI 고정 기준을 따른다.
- 기능 추가보다 핵심 루프 안정화가 우선이다.
- 각 단계는 문서, 코드, 테스트 기준을 함께 갖는다.
- Phase 1 완료 전에는 AI 요약 같은 확장 기능을 넣지 않는다.

## 4. 작업 순서 제안

1. 문서 계약과 실제 응답 필드를 먼저 일치시킨다.
2. 그 다음 서버 validation, 보안 정책, 에러 응답 포맷을 고정한다.
3. 프론트에서 optimistic create, 에러 토스트, 저장 중 제약을 정리한다.
4. 이후 검색 최적화와 폴더 수정 범위를 마무리한다.
5. 마지막에 테스트를 핵심 시나리오 기준으로 확장한다.

## 6. 현행 기준 요약

- 문서 설계와 기본 구현은 이미 완료된 상태다.
- 현재 단계는 신규 스캐폴딩이 아니라 구현 정합성 보정과 품질 보강 단계다.

- 다음 작업은 새 기능 추가보다 계약 정리, validation, 표준 에러 포맷, 보안 정책, UX 안정화, 테스트 확장이 우선이다.
